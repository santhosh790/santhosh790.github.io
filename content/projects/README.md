# Projects

This directory contains MDX files for project case studies. Each file represents a production project with detailed technical documentation.

## File Structure

Each project file should follow this format:

```
slug-name.mdx
```

## Frontmatter Template

```yaml
---
title: "Project Title"
slug: "project-slug"
description: "Brief description"
date: "2024-01-15"
tags: ["Tag1", "Tag2", "Tag3"]
featured: true
published: true
---
```

## Content Structure

Follow the case study format:
1. Business Problem
2. Technical Challenge
3. Architecture
4. Technical Decisions
5. Trade-offs
6. Production Challenges
7. Impact

Projects will be created in Phase 3.
