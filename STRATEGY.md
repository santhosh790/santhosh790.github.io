### Visual Elements

#### Color Palette
**Dark-first approach:**
- Background: Deep charcoal (#0a0a0a - #1a1a1a)
- Text primary: Near-white (#e5e5e5)
- Text secondary: Muted gray (#a0a0a0)
- Accent: Technical blue or green (#00d4ff or #00ff88)
- Code blocks: Darker bg (#0d0d0d) with syntax highlighting

#### Typography
- **Headings:** Inter, SF Pro, or Geist (modern, technical)
- **Body:** Same or System UI stack
- **Code:** JetBrains Mono, Fira Code, or Cascadia Code
- **Size scale:** Large and readable
  - Hero: 3.5-4rem
  - H1: 2.5-3rem
  - H2: 2rem
  - Body: 1.125rem (18px)

#### Spacing & Layout
- **Generous whitespace:** Never feel cramped
- **Max content width:** 1200px (centered)
- **Section spacing:** 120px vertical
- **Grid:** CSS Grid for project cards
- **Responsive:** Mobile-first (320px → 1920px+)

## Key Technical Challenges

### 1. High Latency

Problem

Semantic retrieval + LLM generation exceeded acceptable webpage response time.

Solution

* aggressive caching
* parallel retrieval
* prompt optimization
* smaller embeddings
* asynchronous pipelines
* top-k optimization

Result

Reduced average latency by nearly 40%.
Explanation - [Databricks Blog Post] https://www.databricks.com/customers/eurowings
---

### 2. Vector Search False Positives

Problem

Vector search never returns "no result."

Every query always finds something.

Example

```
"abcd1234"

still retrieves documents.
```

Solution

Implemented confidence scoring using

* cosine similarity threshold
* keyword overlap
* metadata validation
* score-gap heuristic
* hybrid BM25 verification

This became the basis of your Substack article.

Explanation:  https://drsanthosh.substack.com/p/why-vector-search-never-says-no-results
---

### 3. Low Recall

Problem

Correct document existed but wasn't retrieved.

Solution

* Hybrid Search
* better chunking
* metadata filters
* reranking
* synonym expansion
* query rewriting

Explanation: https://www.databricks.com/customers/eurowings
---

## Technical Decisions (WHY)

### Why GPT-4?

* enterprise-grade quality
* multilingual
* Azure compliance
* excellent reasoning

---

### Why Hybrid Search?

Keyword search alone misses semantic intent.

Vector search alone produces false positives.

Combining both significantly improved precision and recall.

---

### Why Databricks Vector Search?

* already using Databricks
* integrated security
* Unity Catalog
* managed infrastructure
* simplified deployment

---

## Trade-offs

### Cost vs Accuracy

Used GPT only for final summarization.

Retrieval remained deterministic.

Reduced token cost substantially.

---

### Latency vs Quality

Limited retrieval to Top-5.

Applied reranking instead of increasing Top-K.

---

### Complexity vs Maintainability

Avoided agentic workflows.

Preferred deterministic pipelines.
